## Implement UI Design with React

### Challenges
- Ensuring all components stayed consistent with the original UI mockups while building them in React
- Managing state across multiple screens (especially for cycle tracking inputs like BBT, CM, and symptoms)
- Making the UI responsive so it works cleanly on both desktop and mobile
- Handling navigation between pages without breaking the user experience

---

### Takeaways
- Breaking the UI into reusable React components (like forms, cards, and navigation) made development much easier and cleaner
- Using props and state correctly is key for keeping the app dynamic, especially for user input and tracking data
- Following the original design mockups closely helped maintain consistency and reduced confusion during development
- Implementing basic client-side validation improved the reliability of user input before sending data to the backend
- Testing on both desktop and mobile layouts is important to ensure a smooth user experience across devices
