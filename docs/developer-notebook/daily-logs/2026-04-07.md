## Designing the Database Schema
### Challenges

### Takeways
- Mongo databases revolve around transactions, so an important thing to look at in designing the database was "what is the user doing?" For Formare, the user is checking or documenting their "body status", that is, what their period is or has been like. So the key transaction is "What is/was my body status today/on X day?" After this, the document and collection set up became apparent.
